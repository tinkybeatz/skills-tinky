# Category: notes

**Scope.** Knowledge-management workflows built on Notion (a tool-cluster around the Notion MCP): capturing conversations/decisions into pages, researching across a workspace, and turning specs into implementation plans. Grouped by the shared tool rather than by function, since they all depend on the Notion MCP connection.

**Naming.** `notion-<function>` — kept even though the folder is `notes/` (the skills are Notion-specific; the folder name is the broader concept). If you add non-Notion note tools later (e.g. Obsidian), consider subfolders.

**Tone & audience.** Knowledge management: well-structured, cross-linked, faithful to sources.

**Requirements.** These need a working **Notion MCP** connection — note this in each skill's description.

**Default frontmatter.**
```yaml
name: <folder-name>
description: <what it does + trigger phrases; mention Notion MCP requirement>
user-invokable: true
```
