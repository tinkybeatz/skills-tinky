## What This Skill Does

Delivers production-ready technical decisions and implementations like an experienced engineering lead (L5-L6 level).

### Core Workflow (5 Steps)

1. **Frame** — Turn ambiguous requests into verifiable engineering questions
2. **Evaluate** — Generate 2+ viable options with explicit trade-offs
3. **Plan** — Break down implementation with acceptance criteria & rollback plans
4. **Execute** — Validation strategy with residual risk ownership
5. **Review** — Confidence assessment backed by evidence

### Output Contract

Every decision includes:

- Executive summary (decision + context)
- Technical reasoning (options + trade-offs)
- Implementation plan (steps, owners, criteria)
- Validation evidence (how we know it works)
- Risks & mitigations (severity-ranked)
- Confidence level (low/medium/high + rationale)
- Next steps (concrete, actionable)

## Usage

### Trigger the Skill

Use **senior-dev** when:

- "How should we approach this decision?"
- "What are our options for solving this?"
- "Is this ready for production?"
- "Who should I mentor among my engineers?"

- ✅ Explicit trade-offs (not hidden)

## Failure Modes & Recovery

| Failure                                                | Recovery                                                                                          |
| ------------------------------------------------------ | ------------------------------------------------------------------------------------------------- |
| "I don't know who to mentor"                           | Ask: readiness (what level?), potential (can they grow?), timeline (how long?)                    |
| "Jane disagrees and I'm stuck"                         | Listen to **her** concerns first (not your objection), find middle ground, pre-align stakeholders |
| "Decision is technically sound but team is misaligned" | Use async RFC-first, presell before meeting, get explicit "I can live with this" not "I love it"  |
| "Output doesn't map to company culture"                | Adjust framing (use company values, not just technical merit), involve company mentors            |
