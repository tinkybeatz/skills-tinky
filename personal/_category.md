# Category: personal

**Scope.** Personal-life and side-project domains that don't fit a functional bucket. This is a *domain* bucket, not "dev work I do off the clock" (a dev skill stays in `dev/` regardless of which project uses it). Organize into one subfolder per domain, each with its own `_category.md`:

```
personal/
  music/      _category.md   <skills>
  home/       _category.md   <skills>
  finance/    _category.md   <skills>
  ...
```

**Naming.** Freeform — whatever reads naturally for the domain.

**Tone & audience.** Whatever fits the domain; can be casual, and can be in French if that's your working language for it. (Unlike the functional buckets, English-first is not required here.)

**Required scaffolding.** Minimal is fine — a single `SKILL.md` is enough. Add `references/` only when useful.

**Default frontmatter.**
```yaml
name: <folder-name>
description: <what it does + trigger phrases>
user-invokable: true
```
