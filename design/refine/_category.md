# Category: design / refine

**Scope.** The design-partner suite: each skill takes an **existing** interface and improves one specific dimension — nothing here builds from scratch (that's `design/build/`). Examples of dimensions: motion, color, clarity, boldness, restraint, accessibility, performance, final polish.

**Naming.** A single verb or adjective naming the transformation — `polish`, `distill`, `bolder`, `quieter`, `colorize`, `harden`, `optimize`. Keep the one-word convention for new additions to this suite.

**Tone & audience.** Senior product designer doing a focused pass. Surgical: change the target dimension, preserve everything else. Respect the project's Design Context.

**Required scaffolding.** Usually single-file (`SKILL.md` only). Add `references/` only if a skill needs shared criteria.

**Default frontmatter.**
```yaml
name: <folder-name>
description: <what it does + trigger phrases>
user-invokable: true
```

**Skills here.** adapt · animate · audit · bolder · clarify · colorize · critique · delight · distill · extract · harden · normalize · onboard · optimize · polish · quieter
